Introducción a HTML

HTML significa HyperText Markup Language, que en español quiere decir "Lenguaje de Marcado de Hipertexto". Es el lenguaje que usamos para crear páginas web. Con HTML puedes definir el contenido de una página como textos, imágenes, enlaces, listas, formularios y mucho más. Todo lo necesario para crear todo tipo de páginas: blogs, tiendas online, portafolios, webs de noticias, etc. Puedes probar páginas HTML en tu ordenador sin conexión a internet, solo necesitas un editor de texto y un navegador para crear un archivo .html y abrirlo directamente.


¿Cuál es la diferencia entre HTML, CSS y JS?

Imagina que estás haciendo un robot:

  • HTML es el esqueleto del robot. Dice qué partes tiene: cabeza, brazos, piernas, ojos, etc. Es la estructura de todo.
  • CSS es el diseño del robot. Decide de qué color son los ojos, si la cabeza es grande o pequeña, si lleva ropa roja o azul. Es el aspecto visual.
  • JavaScript es el cerebro o los movimientos del robot. Gracias a él, el robot puede caminar, hablar, reaccionar cuando le das una orden. Es lo que le da comportamiento, lo que hace que “haga cosas”.

Etiquetas HTML

Una etiqueta en HTML son comandos que usamos para decirle al navegador cómo mostrar el contenido y las diferentes partes del documento.

Son como instrucciones que marcan cada parte del documento. Hay dos tipos principales:


1. Tipos de etiquetas

  • Etiquetas con apertura y cierre:

    • Tienen una etiqueta de apertura y otra de cierre, y definen un bloque de contenido. Ejemplo:

    <h1>Este es un Título Principal</h1>

    <h2>Este es un Subtítulo</h2>

    <p>Este es un párrafo</p>

    <strong>Este texto es importante (negrita)</strong>

    <em>Este texto está enfatizado (cursiva)</em>


  • Etiquetas de auto-cierre o autocontenidas:

    • Son etiquetas que no tienen una etiqueta de cierre correspondiente. Se utilizan para elementos que no contienen contenido, como imágenes o saltos de línea. Ejemplo:


    <img src="foto.jpg" alt="Descripción imagen">

    <img src="logo.png" alt="muestra imagen logotipo">

    <input type="text" name="nombre" placeholder="Escribe tu nombre aquí">

Además, las etiquetas pueden anidarse unas dentro de otras para definir relaciones padre-hijo en el documento. En este caso, la etiqueta interna debe cerrarse antes de la etiqueta externa.


En este ejemplo, la etiqueta "head" contiene dentro otra etiqueta, como "title".

Igual que pasa con "body", que contiene etiquetas como <div> y <script>. Por eso, decimos que:

  • La etiqueta <body> es como el "padre" de las etiquetas <div> <script>.

  • Las etiquetas <div> y <script> son "hijas" de <body>.

  • Las etiquetas <div> y <script> actuarian como "hermanas" porque están al mismo nivel dentro de <body>.

También puedes observar que las etiquetas hijas están un poco más hacia dentro del código (con sangría). Esto se hace siguiendo una regla que usan todos los programadores para escribir el código de forma ordenada. Así, cuando el código crece o cuando alguien más lo lee, es más fácil entenderlo y trabajar con él.


2. Atributos de etiquetas

Las etiquetas pueden llevar atributos, que añaden información extra. Los atributos se colocan dentro de la etiqueta de apertura y se utilizan para configurar propiedades específicas del elemento. Ejemplo:


<a href="https://www.google.com">Ir a Google</a>
  • <a>: define un enlace.
  • href: atributo que especifica el destino.

3. Comentarios

Los comentarios son útiles para agregar notas o aclaraciones en el código sin afectar la visualización de la página. Los comentarios en HTML se escriben así:


<!-- Este es un comentario y no se mostrará en la página -->


Estructura básica de una página WEB

Una página HTML sigue una estructura estándar que incluye varios elementos esenciales para definir el contenido y la configuración de la página.


          <!DOCTYPE html>
          <html>
            <head>
              <meta charset="UTF-8">
              <title>Mi página</title>
            </head>
            <body>
              <h1>Hola Mundo</h1>
              <p>Este es mi primer sitio web.</p>
            </body>
          </html>

Comandos básicos


1. Etiquetas básicas


Etiqueta Apertura/Cierre Función
<!DOCTYPE html>Solo aperturaAunque no es una etiqueta como tal, debe ser la primera línea del archivo HTML. Indica al navegador que use el modo estándar de HTML5 (la versión más actual)
<html lang="es"><html> ... </html>Es el elemento raíz de todo documento HTML. Contiene el resto de las etiquetas de la página y suele incluir el atributo <lang> para indicar el idioma del contenido.
<head><head> ... </head>Contiene metadatos e información sobre el documento, como el título, codificación de caracteres, enlaces a hojas de estilo o scripts. Su contenido no se muestra directamente en la página.
<title><title> ... </title>Define el texto que aparece en la pestaña del navegador cuando se carga la página.
<body><body> ... </body>Alberga todo el contenido visible para el usuario en la página web: textos, imágenes, botones, enlaces, etc.
<h1> ... <h6><h1> ... </h1>Son encabezados o títulos jerárquicos. <h1> representa el título principal y <h6> el de menor nivel. Se usan para estructurar y organizar el contenido.
<p><p> ... </p>Define un párrafo de texto. Es una de las etiquetas más comunes para mostrar información escrita.
<img>Solo apertura (autocierre)Muestra una imagen en la página. Usa atributos como <src> (fuente) y <alt> (texto alternativo).
<a><a> ... </a>Crea un hipervínculo que puede dirigir al usuario a otra página web, documento, sección interna o incluso iniciar descargas.
<div><div> ... </div>Contenedor genérico en bloque que se usa para agrupar elementos y aplicar estilos o scripts.
<span><span> ... </span>Contenedor genérico en línea. A diferencia del <div>, no rompe el flujo del texto y se usa para aplicar estilos a partes específicas del contenido.
<ul><ul> ... </ul>Lista desordenada. Muestra elementos con viñetas. Debe contener elementos <li>.
<ol><ol> ... </ol>Lista ordenada. Muestra los elementos numerados o con letras. También usa <li>.
<li><li> ... </li>Elemento de lista. Se utiliza dentro de <ul> o <ol>.
<header><header> ... </header>Define la cabecera de una página o sección. Suele contener el logo, título y menú de navegación.
<footer><footer> ... </footer>Define el pie de página. Normalmente contiene información de contacto, derechos de autor o enlaces secundarios.
<main><main> ... </main>Contiene el contenido principal del documento. No debe repetirse y excluye barras laterales o menús.
<nav><nav> ... </nav>Define una sección que contiene enlaces de navegación, como menús o barras laterales.
<section><section> ... </section>Representa una sección temática o funcional del contenido. Se suele usar para dividir grandes bloques de contenido relacionados.
<article><article> ... </article>Representa contenido independiente y reutilizable, como una noticia, entrada de blog o comentario.
<aside><aside> ... </aside>Define contenido relacionado pero secundario, como una barra lateral, anuncios o enlaces relacionados.

2. Atributos comunes

Estos son algunos atributos muy utilizados:


Atributo Descripción Ejemplo
href Se usa en enlaces para indicar a qué dirección se dirigirá el usuario al hacer clic. <a href="https://ejemplo.com">Ir al sitio</a>
src Ruta del archivo que se quiere mostrar, por ejemplo una imagen, un video o un script. <img src="imagen.jpg" alt="Descripción">
alt Texto que describe la imagen si esta no se puede mostrar. Mejora la accesibilidad. <img src="gato.jpg" alt="Gato durmiendo">
id Identificador único para un elemento. Se usa para estilos CSS o acceso desde JavaScript. <div id="contenido"></div>
class Permite agrupar elementos que comparten estilo o función. Se puede usar varias veces. <p class="importante">Texto destacado</p>
target Determina dónde se abrirá un enlace. _blank abre en una nueva pestaña. <a href="pagina.html" target="_blank">Ver más</a>
style Permite aplicar estilos CSS directamente en la etiqueta (no recomendado para estilos complejos). <p style="color: blue;">Texto azul</p>
title Muestra un mensaje emergente (tooltip) cuando el usuario pasa el ratón por encima. <abbr title="HyperText Markup Language">HTML</abbr>
type Define el tipo de entrada o contenido. Es muy usado en formularios (text, password, submit...). <input type="password">
value Especifica el valor por defecto de un input, botón, o campo de formulario. <input type="text" value="Juan">
name Asigna un nombre al elemento para identificarlo al enviar un formulario. <input type="email" name="correo">
placeholder Muestra un texto dentro del campo de entrada como ejemplo hasta que el usuario escriba. <input type="text" placeholder="Escribe tu nombre">
disabled Desactiva un elemento del formulario para que no se pueda usar ni editar. <button disabled>No disponible</button>
checked Hace que una casilla o radio aparezca seleccionada por defecto. <input type="checkbox" checked> Acepto</input>
readonly Hace que un campo de formulario se vea, pero no se pueda modificar. <input type="text" value="Solo lectura" readonly>
maxlength Define el número máximo de caracteres que se pueden escribir en un campo de texto. <input type="text" maxlength="10">

Otras funcionalidades útiles

  • Formularios: permiten a los usuarios introducir datos como nombres, correos, contraseñas, opiniones, etc. Estos datos pueden ser enviados a un servidor para ser procesados.

  • Tablas: sirven para presentar información organizada, como horarios, precios, resultados, etc. Están compuestas por filas (<tr>) y columnas (<td>). Por ejemplo, las tablas de etiquetas y atributos de esta página están hechas con este código.

  • Listas ordenadas y desordenadas: se usan para mostrar pasos, elementos o ideas. Las ordenadas (<ol>) se numeran automáticamente, y las desordenadas (<ul>) usan viñetas. Por ejemplo, este listado de otras funcionalidades está hecho con el código de lista desordenada.

  • Buenas prácticas: escribir código limpio y ordenado es fundamental. Algunas recomendaciones son:
    • Usar sangrías para mostrar la estructura jerárquica del código.
    • Escribir comentarios con <!-- comentario --> para explicar partes del código.
    • Nombrar los id y class de forma clara y coherente.
    • Evitar código duplicado o innecesario.
    Esto facilita la lectura del código tanto para ti como para otras personas que lo revisen.
